# UI Aesthetics

Use this workflow to apply the `ui-aesthetics` guidance to a web UI generation, review, or refactor task.

## Load first

- `../../.shared/ui-aesthetics/core.md`
- `../../.shared/ui-aesthetics/reference-map.md`
- `../../.shared/ui-aesthetics/usage.md`

## Then do this

1. Identify the route: generation, review, refactor, component polish, state/motion, or depth/lighting.
2. State a short visual thesis.
3. Preserve scope and information density.
4. Fix composition, spacing, and typography before decorative polish.
5. Keep color, depth, and motion restrained and role-driven.
